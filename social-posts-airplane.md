**Title:** I built an open-source AI assistant and tested it on a plane with no WiFi

**Body:**

Quick story that made my week.

I've been building Molebie AI — an open-source, self-hosted AI assistant — for the past few months. It runs entirely on your own hardware. Local LLM inference, document search (RAG), voice mode, the works. No cloud dependency.

I talk about "privacy" and "local-first" a lot in the README and in my head, but I hadn't really stress-tested the offline story until this week. I was on a flight with no WiFi, opened my laptop, and fired up Molebie AI.

It just worked.

I loaded some documents I needed to review, queried them through the RAG pipeline, and had a normal conversation with the model. All offline. All local. At 35,000 feet.

That moment was the first time the project felt real to me — not as a technical exercise, but as something genuinely useful.

**Where I'm at:**
- v0.1, MIT-licensed, ~16K lines of code
- FastAPI backend + Next.js frontend
- Supports multiple inference backends (Ollama, ONNX, cloud APIs as fallback)
- Built-in RAG with hybrid search, reranking, and contextual retrieval
- Voice mode (STT + TTS)
- Terminal observability dashboard — see exactly what's happening under the hood
